## 第5页：AI核心实现 - RAG技术

### 标题
**RAG知识库检索系统**

### 核心要点
- **知识库分块**：将知识库按段落分割，建立关键词索引
- **相关性检索**：基于关键词交集计算相关性分数
- **Top-K筛选**：返回最相关的3个知识段落
- **动态构建提示词**：将检索结果注入系统提示词
- **智能降级**：无相关知识点时使用基础提示词

### 技术细节
```python
# 关键词提取
keywords = re.findall(r'[\u4e00-\u9fa5]+|[a-zA-Z0-9]+', text.lower())

# 相关性评分
score = len(user_keywords & chunk_keywords)
```

## 附录：技术问题预案

### 可能被问到的问题

#### 1. RAG技术的优势是什么？
**答**：相比传统的关键词匹配，RAG可以：
- 理解用户问题的语义
- 检索最相关的知识片段
- 结合大模型生成准确的回答
- 支持上下文对话

#### 2. 为什么选择DeepSeek而不是GPT？
**答**：
- DeepSeek是国内模型，访问更稳定
- 中文理解能力强
- 成本更低
- 支持商业应用

#### 3. 如何保证AI回答的准确性？
**答**：
- 严格基于知识库，禁止编造
- 明确的提示词约束
- 未知问题引导至人工客服
- 安全第一原则

#### 4. 系统的安全性如何保障？
**答**：
- JWT令牌认证
- bcrypt密码加密
- CORS跨域限制
- API频次限制
- 环境变量管理敏感信息

#### 5. 如何处理高并发？
**答**：
- 异步框架FastAPI
- 数据库连接池
- FAQ缓存机制
- 可水平扩展部署

#### 6. 移动端适配的难点是什么？
**答**：
- 不同屏幕尺寸适配
- 虚拟键盘遮挡输入框
- 触摸事件优化
- 通过响应式设计和JS监听解决
